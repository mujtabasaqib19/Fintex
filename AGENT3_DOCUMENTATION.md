# 🤖 Agent 3: Real-Time Market Agent for Pakistan Stocks

## Overview

Agent 3 is a specialized retrieval agent for Pakistan Stock Exchange (PSX) stock market data. It retrieves live stock prices, market movements, and trading data specifically from the `stock_prices` table in Supabase.

**Context**: Pakistan Finance Expert (not global)

## What Agent 3 Retrieves

- ✅ Live stock prices (OHLC - Open, High, Low, Close)
- ✅ Historical price data
- ✅ Trading volumes
- ✅ Market statistics and analysis
- ✅ Price trends and movements
- ✅ Stock comparisons
- ✅ Market snapshots

## Database Schema

Agent 3 queries the `stock_prices` table in Supabase:

```sql
create table public.stock_prices (
  id bigint generated by default as identity not null,
  symbol text not null,
  date date not null,
  open numeric(10, 2) null,
  high numeric(10, 2) null,
  low numeric(10, 2) null,
  close numeric(10, 2) null,
  volume bigint null,
  created_at timestamp with time zone not null default timezone ('utc'::text, now()),
  constraint stock_prices_pkey primary key (id),
  constraint stock_prices_symbol_date_key unique (symbol, date)
) TABLESPACE pg_default;

-- Indexes for performance
create index idx_stock_prices_symbol on public.stock_prices using btree (symbol);
create index idx_stock_prices_date on public.stock_prices using btree (date);
create index idx_stock_prices_symbol_date on public.stock_prices using btree (symbol, date);
```

## API Reference

### Core Methods

#### `get_latest_price(symbol: str)`
Get the most recent price data for a stock.

**Returns**: Dict with OHLC data, volume, and date

#### `get_price_history(symbol, start_date, end_date, limit=100)`
Get historical price data.

**Returns**: List of price records ordered by date

#### `get_price_stats(symbol, days=30)`
Calculate statistical metrics over a period.

**Returns**: Dict with min, max, avg, change%, trend, volume stats

#### `get_volume_analysis(symbol, days=30)`
Analyze trading volume patterns.

**Returns**: Dict with volume statistics

#### `compare_stocks(symbols, days=30)`
Compare performance of multiple stocks.

**Returns**: List of stats for each stock, sorted by performance

#### `get_top_movers(limit=10, days=7, direction='up')`
Get top gaining or losing stocks.

**Returns**: List of top movers with stats

### Formatting Methods

All methods return formatted strings suitable for LLM context:

- `format_price_for_context(price_data)` - Single price record
- `format_history_for_context(history, symbol)` - Historical data
- `format_stats_for_context(stats)` - Statistical analysis
- `format_comparison_for_context(comparisons)` - Multi-stock comparison

## Performance Considerations

1. **Indexing**: Ensure indexes exist on `symbol` and `date` columns
2. **Caching**: Consider caching frequently accessed stocks
3. **Limits**: Use appropriate limits when fetching multiple stocks
4. **Batch Operations**: Use `get_multiple_latest_prices()` for efficiency

## Troubleshooting

### No data returned
- Verify the symbol exists in database
- Check date ranges are valid
- Ensure data has been ingested

### Slow queries
- Check if indexes exist
- Reduce date ranges
- Limit number of symbols

### Integration issues
- Ensure stock_prices table exists
- Verify Supabase connection is active
- Check .env configuration
